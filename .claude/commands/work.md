# 개요

아래의 내용에 따라 새로운 마일스톤 m12를 Generate_task.md 파일에 추가하고 m12-start.md 파일을 작성한다.

## 관리자 페이지 수정

관리자 페이지는 구현되어 있는데 다이렉션이 전혀 되지 않는 것으로 보인다. 문제 확인 후 수정이 필요하다.

## 프로필 페이지 수정

- 학생의 기본 정보(이름, 소속 학교, 학년, 반, 학생 코드 등)를 프로필에 표시한다. 회원가입 API나 DB에 해당 정보가 없는 경우 이 부분도 수정한다.

## 영양사 UI 및 기능 수정 사항

### 생성 조건 입력 이전에 해야 할 일

- 소속 단체(학교) 정보를 프로필에서 확인할 수 있도록 한다.
  - 학생 / 교직원 / 보호자는 프로필 화면에서 소속 단체 정보를 확인하거나 수정 및 삭제 가능하도록 한다.
  - 영양사는 대시보드에서 학교의 정보(학교 이름, 성별 / 학년별 인원 분포)
- 영양사 대시보드와 수요 집계 대시보드를 합친다. 수요 집계 대시보드의 보기를 우선으로 해서 합친다.

### 생성 조건 항목

- 다음 항목들을 기본값으로 지정한다. 일 평균 제공량을 입력하도록 지시하고, 식단을 편성할 때는 주 단위 총합이 기준치에 오차 범위 +-10퍼센트 이내가 되도록 계산한다.
  - 칼로리: kcal 단위, 소속 단체의 정보를 활용해 남녀 및 연령 분포를 이용해 필요 섭취량을 가중평균한 값을 초기값으로 지정한다. 표는 다음과 같이 주어지므로 이에 따라 계산할 것.
    | 연령(세) | 에너지필요추정량 남자 (kcal/일) | 에너지필요추정량 여자 (kcal/일) |
    |---|---:|---:|
    | 1~2 | 900 | - |
    | 3~5 | 1400 | - |
    | 6~8 | 1700 | 1500 |
    | 9~11 | 2000 | 1800 |
    | 12~14 | 2500 | 2000 |
    | 15~18 | 2,700 | 2,000 |
    | 19~29 | 2,600 | 2,000 |
    | 30~49 | 2,500 | 1,900 |
    | 50~64 | 2,200 | 1,700 |
    | 65~74 | 2,000 | 1,600 |
    | 75 이상 | 1900 | 1500 |
  - 탄수화물: 일일 에너지에 대한 비율(퍼센트 단위)로 입력, 60을 초기값으로 지정
  - 단백질: 일일 에너지에 대한 비율(퍼센트 단위)로 입력, 20을 초기값으로 지정
  - 지방: 일일 에너지에 대한 비율(퍼센트 단위)로 입력, 20을 초기값으로 지정
  - 칼슘: mg 단위, 300을 초기값으로 지정
  - 나트륨: mg 단위, 400을 초기값으로 지정
- 버튼을 이용해 입력 항목 추가 및 삭제가 가능하게 한다. 예를 들어, 항목 추가 버튼을 눌러 비타민 C 항목을 추가하고, 항목 삭제 버튼을 눌러 지방 항목을 제거할 수 있게 한다. 새롭게 추가하는 항목은 단위를 직접 입력하도록 지시한다.
- NEIS 정보는 식단 생성 탭 로드 시 미리 api를 가져와서 리스트 박스로 학교 이름, 코드 등을 선택 또는 검색할 수 있도록 한다.
- 급식 단가는 월/주/일 & 평균/총합 선택지를 주고 입력받도록 한다. 메뉴별 단가는 NEIS API를 이용해 해당 학교의 메뉴 단가를 이용해 입력받은 단가 이내로 편성한다.
