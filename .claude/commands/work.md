# 개요

아래의 내용에 따라 새로운 마일스톤을 생성해 Generate_task.md 파일에 추가한다.

# 버그 목록

## 1순위(해결 필수)

- 대체 식단을 메뉴별로 만들도록 되어 있는데, 끼니별로 만들도록 구성해야 한다. 끼니별로 섹션을 구성하고, 모달 폼을 활용해 대체식단을 추가한다. 대체식단 생성 폼은 식단 수동 입력 폼과 같은 형식이어야 한다. 저장 버튼을 클릭하면 추가된 대체식단이 하나인 경우 그대로 저장하고, 둘 이상인 경우 설문을 자동으로 생성하는 로직으로 동작해야 한다.

- AI 식단 생성 결과를 끼니별 카드 형태의 미리보기로 표시하고 영양사가 결과를 수동으로 편집한 뒤 최종 저장 버튼을 눌러 저장할 수 있어야 한다.

- MealPlanPage의 식단 메뉴 수동 입력 폼은 각 메뉴별로 포함 식재료를 콤마로 나누어 입력하는 칸이 추가로 필요하다. 또, 알레르기 자동 태깅에 메뉴별 매핑 외에도 식재료 매핑이 있어야 한다. 두 매핑을 or 조건으로 합쳐 태깅한다.(보수적으로 태깅)

- 대체 식단 추가 폼, AI 식단 생성 결과 편집 폼, 식단 메뉴 수동 입력 폼은 무조건 같은 입력 필드들을 공유하도록 구성해야 한다.

- 회원정보와 회원가입할 때 학년이 1~12로 되어 있고 반 수도 실제 학교 정보랑 일치하지 않으므로 Organization 테이블의 gradeStructure 필드와 연동해 올바르게 적용될 수 있도록 해야 한다.

## 2순위(1순위 해결 후 시작)

- 이메일과 웹 푸시 알림이 실제로 동작하는지 확인이 필요하다.

- pdf 저장 외에 엑셀 파일로 저장하는 기능이 필요하다. 월별 식단을 엑셀 표로 저장할 수 있어야 한다.

- MealPlanPage에서 영양사의 경우 임시저장된 식단이 존재하는 날짜가 표시되어야 한다. 또, 선택 버튼을 만들어 클릭하면 각 날짜에 체크박스가 표시되고 체크한 날짜 중에 임시저장된 식단이 존재하는 경우 일괄 저장할 수 있는 기능을 추가한다.

- 현재 학생, 교직원, 보호자, 영양사 로그인 탭 구분이 전혀 안 되고 있다. 예를 들어 학생 탭에서 영양사 계정 로그인 시 404가 뜨는 것이 원하는 로직이다.

- MealPlanpage의 AI 초안 생성 버튼이 비활성화 상태이다. 캘린더 및 슬라이더 보기에서 날짜를 선택할 경우 AI 초안 생성 버튼이 활성화되도록 한다. 버튼을 클릭하면 AIMealPlanPage로 이동한 뒤 선택한 기간이 지정된 상태로 로드한다. 연속되지 않은 날짜가 선택되어 있는 경우 선택한 모든 날짜의 첫 날짜와 마지막 날짜로 지정한다.

- AIMealPlanPage에 식단 생성에 주말을 포함하는지 여부를 묻는 질문을 추가한다. AI 프롬프팅도 함께 적절히 수정한다.

- User의 studentCode 필드는 비즈니스 로직에 직접 관여하지 않으니 제거한다. 연결된 회원가입 폼의 학번 입력 필드도 함께 삭제한다.

## 3순위(UI/UX 선택 사항)

- UI 개선 사항: 로고를 사이드바가 아니라 네비게이션 바에 포함되도록 한다. 네비게이션 바를 상단에 두고 사이드바가 네비게이션 바 밑으로 위치하도록 한다. 그리고 네비게이션 바는 스크롤해도 화면 상단에 고정되어 있도록 한다.

- 비밀번호 표시 옵션을 추가한다. (눈 버튼을 누르면 마스킹이 해제되도록)
